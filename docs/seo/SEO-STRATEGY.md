## 1. ICP-Based Keyword Strategy

### Target Customer Profile

Vantro Digital serves small businesses generating **$300K–$1.5M in annual revenue** — typically owner-operated service businesses that want growth but don't have in-house marketing staff.

**Primary ICP Verticals:**

| Vertical | Pain Points | Search Behavior |
|---|---|---|

## 6. Technical SEO Checklist

### Core Web Vitals (React/Vite SPA)

**Largest Contentful Paint (LCP) — Target: < 2.5s**

- [ ] Hero image must be an `<img>` tag with `fetchpriority="high"` — NOT a CSS background image
- [ ] Add `<link rel="preload" as="image">` for the LCP image in `index.html`
- [ ] Serve images in **WebP** format; use `<picture>` with AVIF fallback
- [ ] Enable Vite's `build.rollupOptions` for code splitting; lazy-load below-fold components
- [ ] Use a CDN with edge caching (Cloudflare, Vercel Edge, or Netlify CDN)
- [ ] Fonts: use `font-display: swap`; preconnect to Google Fonts (`<link rel="preconnect">`)

**Cumulative Layout Shift (CLS) — Target: < 0.1**

- [ ] Set explicit `width` and `height` on all images and video iframes
- [ ] Reserve space for dynamic content (ads, embeds) with `min-height`
- [ ] Avoid inserting content above existing content after load
- [ ] Use skeleton loaders instead of content popping in

**Interaction to Next Paint (INP) — Target: < 200ms**

- [ ] Defer non-critical JavaScript with `<script defer>` or `React.lazy()`
- [ ] Break up long tasks (>50ms) in the main thread
- [ ] Minimize third-party scripts (chat widgets, analytics) — load after user interaction where possible
- [ ] Use `useCallback` and `useMemo` for expensive React renders

---

### React SPA-Specific SEO Requirements

- [ ] **Pre-rendering / SSR:** Vite SPAs are not server-rendered by default. Implement one of:
  - **Vite Plugin SSG** (`vite-plugin-ssg`) — generates static HTML at build time (recommended)
  - **React Router with SSR** — if using React Router 6.4+ with data loaders
  - **Prerender.io** — dynamic rendering fallback (last resort)
- [ ] Verify Googlebot can crawl JS-rendered content via Google Search Console → URL Inspection
- [ ] Each route must render unique `<title>`, `<meta description>`, and `<canonical>` tags
- [ ] Use `react-helmet-async` or `@tanstack/react-start` for dynamic `<head>` management
- [ ] Ensure `<noscript>` fallback content exists for critical pages

---

### Additional Technical Checks

- [ ] **Canonical tags:** Every page must have a self-referencing canonical `<link rel="canonical" href="...">`
- [ ] **HTTPS:** Verify all pages load over HTTPS; no mixed content warnings
- [ ] **404 handling:** Custom 404 page must exist and not return a 200 status code (SPA pitfall — configure on hosting layer)
- [ ] **Redirects:** All `http://` → `https://`, all `www.` → non-www (or vice versa), enforce consistently
- [ ] **Page speed:** Run Lighthouse in CI; target 90+ Performance score on mobile
- [ ] **Image optimization:** All images compressed; use lazy loading (`loading="lazy"`) for below-fold images
- [ ] **Open Graph tags:** Every page needs `og:title`, `og:description`, `og:image`, `og:url`
- [ ] **Twitter Card tags:** `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`
- [ ] **Breadcrumbs:** Implement on blog and inner pages (also add BreadcrumbList schema)
- [ ] **Internal linking audit:** Every blog post must link to at least one service page; service pages link to relevant blog posts
