## 6. Technical SEO Checklist for Vite React SPA

Single-Page Applications (SPAs) built with React and Vite present unique SEO challenges. Crawlers may view client-side-rendered JavaScript as an empty `<div id="root"></div>` if not properly pre-rendered. This checklist guarantees search engines read static HTML with accurate metadata per route.

```
                    Vite React SPA Technical Pipeline
┌──────────────────┐    ┌────────────────────┐    ┌─────────────────────┐
│  React Source    │───>│  vite-react-ssg /  │───>│ Static Pre-rendered │
│  (React Router)  │    │  Prerender Build   │    │ HTML Files per Route│
└──────────────────┘    └────────────────────┘    └─────────────────────┘
                                                             │
                                                             ▼
                                                  ┌─────────────────────┐
                                                  │ Unique Head, Title, │
                                                  │ Meta & JSON-LD Tags │
                                                  └─────────────────────┘
```

### 1. Pre-Rendering Strategy (CRITICAL)

* [ ] **Implement Build-Time Pre-Rendering / Static Site Generation (SSG):**
  * Use `vite-react-ssg` or `vite-plugin-prerender` to crawl dynamic routes during `vite build` and output individual static `.html` files for every route (`/`, `/services/web-design`, `/services/seo`, `/blog/post-title`, etc.).
  * *Alternative Option:* Integrate a dynamic pre-rendering service (such as Prerender.io) at the edge/hosting layer (Netlify/Vercel/Cloudflare Workers) to serve pre-rendered HTML to user-agents containing `googlebot`, `bingbot`, `yandex`, `duckduckbot`, `twitterbot`, `facebookexternalhit`.
* [ ] **Verify Static Render Output:** Run `curl -A "Googlebot" https://vantrodigital.com/services/seo` and confirm that full HTML content, H1 tags, paragraph text, and metadata are returned without needing JavaScript execution.

### 2. Dynamic Head & Metadata Management

* [ ] **Install & Configure `react-helmet-async`:**
  * Wrap the application root in `<HelmetProvider>`.
  * Define unique `<title>`, `<meta name="description">`, and `<link rel="canonical">` for every single page route.
* [ ] **OpenGraph & Twitter Card Setup:** Ensure each page includes `og:title`, `og:description`, `og:image`, `og:url`, `og:type="website"`, and `twitter:card="summary_large_image"`.

### 3. XML Sitemap & Robots.txt Protocol

* [ ] **Automated Sitemap Generation:**
  * Configure `vite-plugin-sitemap` in `vite.config.js` to automatically generate `dist/sitemap.xml` upon build, including all static routes, service pages, case studies, and blog post URLs.
  * Submit `https://vantrodigital.com/sitemap.xml` to Google Search Console and Bing Webmaster Tools.
* [ ] **Robots.txt Directive (`/public/robots.txt`):**
```txt
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/
Disallow: /draft/

Sitemap: https://vantrodigital.com/sitemap.xml
```

### 4. Canonicalization & Clean URLs

* [ ] **Strict Canonical Enforcement:** Every page must self-reference its clean, lowercase URL without trailing slashes or URL query parameters via `<link rel="canonical" href="https://vantrodigital.com/services/web-design" />`.
* [ ] **HTTPS & Non-WWW Redirects:** Ensure server/hosting configuration forces `http://` -> `https://` and `www.vantrodigital.com` -> `vantrodigital.com` 301 redirects.

### 5. Core Web Vitals & Asset Performance

* [ ] **Largest Contentful Paint (LCP) < 2.0s:**
  * Convert all hero images and project screenshots to `.webp` or `.avif` format.
  * Preload primary hero assets using `<link rel="preload" as="image" href="...">`.
  * Explicitly set `width` and `height` attributes on all `<img>` tags to eliminate Layout Shift (CLS).
* [ ] **Interaction to Next Paint (INP) < 150ms:**
  * Defer non-critical third-party scripts (Google Tag Manager, analytics scripts, chat widgets) using `async` or `defer`.
* [ ] **Font Display Optimization:** Use `font-display: swap;` for all Google Fonts / web fonts to prevent FOIT (Flash of Unseen Text).

### 6. Structured Data (JSON-LD) Deployment

* [ ] **Global Organization & WebSite Schema:** Inject global `Organization` and `WebSite` schemas into root layout.
* [ ] **Page-Specific Schema:** Inject `Service` schema on core service pages, `BlogPosting` and `Person` (Author) schemas on articles, and `FAQPage` schema on pages containing collapsible FAQs.
